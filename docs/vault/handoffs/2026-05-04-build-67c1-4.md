## What shipped this session

### Track 1 — Test data cleanup in AAA prod (no commit; DB state change only)

Cleanup transaction ran via Supabase MCP `execute_sql`. Two corrections vs the prepared SQL block in [[2026-05-04-build-67c1-3]]:

- **Preset swap order.** Handoff SQL did `UPDATE old=true; UPDATE new=false`. The partial-unique index `idx_pdf_presets_org_default WHERE is_default = true` is non-deferrable in Postgres (checked end-of-statement, not end-of-transaction), so promoting the old default while the new is still `is_default=true` would fail at statement 1. **Fix:** demote new first, then promote old.
- **Estimate / invoice order.** Handoff SQL did `DELETE invoices` then `UPDATE estimates`. Both FKs (`estimates.converted_to_invoice_id` → `invoices(id)` and `invoices.converted_from_estimate_id` → `estimates(id)`) are NO ACTION (default). Deleting the invoice while the estimate still points to it would fail. **Fix:** UPDATE the estimate first (clears the pointer), then DELETE the invoice.

Final transaction (executed):

```sql
BEGIN;
UPDATE pdf_presets SET is_default = false WHERE id = '19389f87-f361-417d-aa54-46a021b84e8e';  -- demote Test 3 Preset
UPDATE pdf_presets SET is_default = true  WHERE id = '0578b56c-90c2-4cb6-8998-894a761f8e67';  -- promote Estimate (default)
DELETE FROM pdf_presets WHERE id = '19389f87-f361-417d-aa54-46a021b84e8e';                    -- delete Test 3 Preset
UPDATE estimates SET status = 'sent', converted_to_invoice_id = NULL, converted_at = NULL
  WHERE id = '500f52f2-c34b-4aaf-94aa-06753606b330';                                          -- restore EST-7 (clears FK)
DELETE FROM invoices WHERE id = '6970525b-eddc-46f4-846a-3eb4576263fb';                       -- delete INV-3 (CASCADE handles invoice_sections + invoice_line_items)
COMMIT;
```

Post-commit verification: `Estimate (default)` is the sole estimate preset for AAA org (is_default=true), `WTR-2026-0018-EST-7` back to `sent` with NULL pointer, `WTR-2026-0018-INV-3` does not exist, no orphan "Test 3 Preset" row.
