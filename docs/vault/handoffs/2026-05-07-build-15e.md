  - **`/sign/[token]` page 500'd in real browsers.** Root cause: `react-pdf` 10 imports `pdfjs-dist` at module-eval time, which throws under Next 16 Server-Component SSR. Bisected over four debug commits (`c8c25a9`, `dc4cc5b`, `5005e96`, `dc8bb66`) тАФ stub-by-stub elimination of `<SignedRedirectWrapper>`, `<PdfCanvas>`, `<SignaturePadModal>` to confirm `<PdfCanvas>` was the culprit.   - **`/pdf.worker.min.mjs` redirected to `/login` by the auth proxy.** `src/proxy.ts` (Next 16 renamed `middleware` тЖТ `proxy` already on 2026-04-22) matcher only exempted image extensions in its negative lookahead; `.mjs` got caught and 307'd to `/login`. Browsers cached the redirect aggressively. Fixed at `09aec43` by adding `pdf\.worker\.min\.mjs` to the negative-lookahead alternation. Cache-buster `?v=${pdfjs.version}` added at `397c96c` to force a fresh fetch past the cached 307. Debug `onLoadError` log added at `3b5d3d4`.
  - **PDF still hung on "Loading PDFтАж" after the proxy fix because pdf.js's fake-worker fallback iframe was sandboxed without `allow-scripts`** тАФ 28├Ч console errors of the form "Blocked script execution in '<URL>' because the document's frame is sandboxed and the 'allow-scripts' permission is not set." The real worker was failing to instantiate because of the cached 307; pdf.js fell back to its fake-worker path which spins up a sandboxed iframe to eval the worker module client-side, and the sandbox attribute it sets blocks script execution. Fixed at `5241dbb` by switching `configurePdfjs()` (in `src/lib/pdf/configure-pdfjs.ts`) from a `/pdf.worker.min.mjs?v=...` URL to bundler-resolved `new URL("pdfjs-dist/build/pdf.worker.min.mjs", import.meta.url).toString()`. Webpack/Turbopack picks up that pattern and emits the worker into `_next/static/chunks/...mjs`, which is auth-proxy-exempt (already in the matcher) AND content-hash-busted by build hash AND avoids the `/public/` postinstall-hook fragility entirely. The fake-worker fallback never runs because the real worker now instantiates cleanly.
