## Notes for next session

- **CLI archive is the right default for iOS builds going forward.** `xcodebuild -project ios/App/App.xcodeproj -scheme App -configuration Release -destination 'generic/platform=iOS' -archivePath build/Nookleus-build{N}.xcarchive -allowProvisioningUpdates archive` is reproducible, fast (~2-3 min), and emits a clean exit code. Wrap in `run_in_background: true` for the long step. The `-allowProvisioningUpdates` flag matters: without it, automatic signing can fail on a fresh team-membership check. Distribution still needs Organizer (or an ASC API key); the archive itself does not.

- **Build numbers live in `pbxproj`, not `Info.plist`.** The Info.plist references `$(CURRENT_PROJECT_VERSION)` and `$(MARKETING_VERSION)` as variables; the actual integers are in `ios/App/App.xcodeproj/project.pbxproj` under both Debug and Release configurations. Both occurrences must be bumped (`replace_all: true` on the exact line). TestFlight rejects duplicate build numbers, so this is a hard prerequisite for every upload.

- **The offline-stub workflow is fragile.** Today: `out/` is gitignored, but cap sync requires `out/index.html` to exist before it'll copy a stub into `ios/App/App/public/`. If a fresh clone runs cap sync without first creating `out/index.html`, the iOS bundle will lack an offline fallback. Two cleaner long-term options: (1) commit a minimal `out/index.html` stub but exclude the rest of `out/*`; (2) add a small npm script (e.g. `prepare:ios`) that writes the stub before cap sync. Option (1) is the smaller change. Option (2) is more flexible if the stub ever grows. Today's session created the stub by hand; it'll vanish on the next clone.

- **Camera-preview + filesystem are present in node_modules but not declared in package.json.** This is a real package-manager hygiene gap. `b388106` modified Package.swift directly; the next maintainer (or cap sync) won't know those plugins were intended without the npm declaration. Fix at the start of 65b before any other iOS work.

- **Vanessa's MacBook is now a known iOS-build machine.** Hostname `Vanessas-MacBook-Pro.local`. Eric's Apple ID + signing key are present and working. Past sessions referenced "the Mac" generically; this is now identifiable. The 00-glossary "Machines and environments" section may want a one-line entry but isn't critical.

- **GUI Organizer auth flow is fine for this volume.** No need to set up an ASC API key just for occasional manual TestFlight uploads. Set one up if/when CI/CD or scheduled uploads come into scope.
