### Two findings out of the manual pass

1. **Permissions admin-bypass** at `permissions-api.ts:44` — `if (membership.role === "admin") return ok: true` short-circuits before the permission check. First attempt at Test 11 (toggling `manage_pdf_presets.granted = false` while keeping role=admin) didn't trigger 403s because admins always pass. Second attempt with role=`crew_lead` + revoked permission worked. **This is by design**, but counter-intuitive — documented in the test results doc.

2. **Settings page UX gap** — `/settings/pdf-presets` renders the "+ New Preset" / "Set as default" / "Edit" buttons regardless of permission. Clicking yields a 403 toast (with the M2 fix surfacing the route's error message). Security boundary is intact at the route layer; UX is just worse than the spec's "page denies" implied. Filed as a follow-up chip — most settings pages in this build follow the same pattern.

**Pattern reinforced this session:**
- **Transactional DB-level testing** (DO-block + RAISE EXCEPTION rollback) is invaluable for testing destructive flows on prod data. Test 12 caught the latent 67b bug without polluting prod.
- **Permissions admin-bypass is a design pattern, not a bug.** When testing role-gated endpoints, you must demote the role AND revoke the permission. Toggling the permission row alone does nothing for admins.
- **Latent bugs hide in untaken paths.** The `invoice_line_items.code` column has been missing since 67b cleanup (4 days). It only surfaced once a real conversion ran. **Lesson:** when you carry-forward an RPC literally, also carry-forward the schema assumptions and re-test the actual code path against actual data, not just the function signature.
- **Pre-flight pattern keeps paying off.** Same pattern as 67b session 9 — read the codebase before assuming the plan's named imports + signatures + table columns exist. The handoff for [[2026-05-04-build-67c1-2]] caught 12 plan-codebase divergences in the T1–T17 phase; this session caught and fixed 1 more (the T22 plan options didn't match codebase reality).
