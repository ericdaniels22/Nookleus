  seeded temp table (`begin; … rollback;`).

## How it was built — TDD

6 red→green cycles, run **against the live AAA-prod database** via the Supabase
MCP `execute_sql` (a read-only / temp-table test harness — there is no local
Postgres, no `psql`/`docker`/`supabase` CLI, and scratch Supabase is paused per
`project_scratch_supabase_paused`):

1. bare 10-digit → `+1XXXXXXXXXX`
2. 11-digit country code → `+1XXXXXXXXXX`
3. backfill rewrites a parseable row
4. backfill **preserves** unparseable rows — this RED caught a real data-loss
   bug: an unguarded `UPDATE` set unparseable values to `NULL`. Fixed with a
   `norm() IS NOT NULL` guard.
5. backfill stamps `updated_at` on changed rows
6. backfill skips already-canonical rows (idempotent — no needless write)

Then a regression sweep of all 11 `normalizePhoneToE164` edge cases — full
parity. The function/UPDATE were also dry-applied on prod inside a rolled-back
transaction before the file was finalised.
