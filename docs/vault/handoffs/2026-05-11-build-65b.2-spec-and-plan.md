## Decisions locked (full spec)

Comprehensive list — exhaustive because the plan references all of these.

### Layout

- **Scope:** hybrid (visual language matches reference; Nookleus-load-bearing features kept and restyled). Eric picked option C of three.
- **Orientation:** portrait only. `Info.plist` currently allows landscape; needs locking.
- **Viewport:** letterboxed, **4:3 aspect ratio**, rounded corners (~16pt). Eric added the 4:3 spec mid-grill.
- **Bezel color:** flat Nookleus brand green `#0F6E56` (= `--brand-primary`, same shade as the active sidebar nav tab). Eric chose this over my navy + green-accent recommendation; he wanted brand-forward over color-fidelity-in-photo trade-off.
- **Top strip:** ~90pt tall, extends to top edge of screen, iOS status bar overlays on green (one continuous strip).
- **Bottom strip:** ~150pt tall (asymmetric — bottom has more controls).
- **Parent app chrome (AAA "Disaster Recovery" logo + notification bell)** suppressed while camera is open. Fix: add `/^\/jobs\/[^/]+\/capture(\/|$)/` to `INTERNAL_FULLSCREEN_PATTERNS` in `src/components/app-shell.tsx:15-17`.

### Top strip layout (final order)

- **Top-left, alone:** X (close) button.
- **Top-right cluster** (left to right): Flip camera, Flash, Mode toggle, Settings gear.
- All four cluster icons white on green.

### Bottom strip layout (final order)

- **Above the action row:** big count number, centered, white on green. Hidden when count = 0.
- **Action row** (left to right): Queue button, Shutter, Done pill.

### Controls behavior

- **X close:** confirms when count > 0 with "Leave camera? Your N photos will still upload." Yes/No (Stay/Leave). Silent exit at count = 0. Eric chose Option B against my Option A (silent always) recommendation — wants the safety message even though captures aren't actually destroyed.
- **Flash:** 3 modes cycle — off → on → flashlight (`flashMode: 'torch'`) → off. The `@capacitor-community/camera-preview` plugin already supports `'torch'`; verified via `node_modules/@capacitor-community/camera-preview/dist/esm/definitions.d.ts:52` (`CameraPreviewFlashMode = 'off' | 'on' | 'auto' | 'red-eye' | 'torch'`).
- **Mode toggle:** 2-state swap (Tag ↔ Rapid). **Default = Tag.** Currently `'rapid'` per `src/lib/mobile/use-capture-mode.ts:7` — needs flip. Icon swaps to show current state (lucide `Tag` icon vs `Zap`). 2-state, not future-proofed N-state per Eric's call.
- **Settings gear:** opens placeholder sheet ("Settings will appear here in a future update."). Eric chose ship-with-placeholder over my cut-it-for-now recommendation.
- **Shutter:** solid white circle with thin lighter ring outline, ~80pt diameter. Press behavior: `active:scale-95` only. **No full-screen flash** — Vanessa shoots 100-shot bursts; full-screen flash 100× = migraine.
- **Done:** pill labeled "Done" (text only). No icon, no count badge. Always visible regardless of count. Eric chose count-separate over my count-on-Done recommendation.
- **Queue button:** stack/list icon + status dot. No dot = idle, amber = uploads in flight, **red pulsing** = at least one failure. Eric chose pulsing dot over my one-shot banner recommendation (banner is the louder Finding-B prevention signal; pulsing dot is the subtler version Eric preferred).

### Empty state (count = 0)

- Count number hidden.
- Done + Queue buttons visible (Queue may still show dot from prior session).
- No layout shift — count fades in once first capture lands.

### Tag-after flow

- **Out of scope for build 65b.2.** Apply minimal visual restyle only during implementation (swap `bg-black/80` to brand-green palette, restyle Continue button to white-with-green-text).
- Full redesign deferred to its own future grill (own design tree: bottom-sheet vs full-screen preview, tag picker UX, caption affordance, gesture handling, skip/persist semantics).

### Parking-lot items, confirmed

- Flash "flashlight" = LED stays on continuously (`'torch'` mode), not capture-triggered: **yes**.
- Shutter press behavior: briefly shrinks, no full-screen flash: **yes**.
- Icon/text color on green bezel = white: **yes**.
- Haptic vibration on first upload failure: **skip**.
- Permission-denied screen: restyle to green theme (don't full-redesign): **yes**.

- Banner/toast on upload failure (Eric chose pulsing dot only).
