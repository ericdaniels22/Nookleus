### Track 2 — T17 inline fix landed (`5144122`)

**Symptom:** At Test 1 step 1, modal opened with subject reading literally `Estimate from {company_name} — {job_address}` and body still containing `{customer_first_name}`. The preview API returned `unresolvedFields: []` even though no merge fields were resolved.

**Root cause:** Plan-codebase divergence missed during T4 review. The plan's seeded templates used single-brace `{field}` syntax with field names `{job_address}` and `{customer_first_name}`. The existing `applyMergeFieldValues` helper in `src/lib/contracts/merge-fields.ts` only matches `{{double_brace}}` (Tiptap pill or bare `{{token}}`) and only knows fields declared in `MERGE_FIELDS`. Single-brace strings flowed through verbatim AND `job_address` / `customer_first_name` aren't `MERGE_FIELDS` members.

**Fix scope (4 layers):**
1. **Code:** `src/lib/contracts/merge-fields.ts` — added `customer_first_name` to `MERGE_FIELDS` (Customer category) + assigned it from `contact.first_name` in `buildMergeFieldValues`. 2-line diff.
2. **Migration file:** `supabase/migration-build67c2-send-via-email.sql` — rewrote both the backfill `UPDATE` block (line 32–38) AND the `seed_default_payment_email_settings` trigger function (line 85–88) to use `{{double_brace}}` syntax with field names that exist (`{{customer_first_name}}`, `{{property_address}}`, `{{company_name}}`). Added a comment explaining the divergence so the rationale isn't lost.
3. **Live data (out-of-band SQL):** `UPDATE payment_email_settings SET <4 templates> = …` — corrected templates landed for both AAA + Test Co; `CREATE OR REPLACE FUNCTION public.seed_default_payment_email_settings(...)` so the trigger seeds future orgs with the corrected templates.
4. **Live setting:** Set `company_settings.value='Test Company'` for Test Co's `key='company_name'` (was empty); without this `{{company_name}}` would have rendered as the unresolved-blank span and Test 1's resolved-subject verification would have looked weird.

Post-fix verification: `/api/estimates/.../send/preview` returns `subject: "Estimate from Test Company — 67c2 Send-Test Boulevard"` + body with `Hi TestPass, Attached is the estimate for the work at 67c2 Send-Test Boulevard…` — fully resolved.

This fix is also the answer for any other module that consumes the same email-template seeds (none currently exist, but future builds adding sub-templates should follow the `{{double_brace}}` + `MERGE_FIELDS`-vetted-field-names pattern).
