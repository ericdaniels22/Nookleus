# RoomPlanCapture — Xcode setup

Issue **#863** (PRD #859, Sketch slice **S11**) delivers the **native RoomPlan
capture plugin**: a Swift Capacitor plugin that scans a room on a LiDAR device
and returns the geometry the Sketch needs. Per [ADR 0025] a scan is an *input*
to the one parametric Sketch, not a parallel artifact — so this plugin returns
the `CapturedRoom` payload and a mesh reference; **mapping that payload onto the
Sketch's Room model and the review/correct flow are a separate slice.**

## Delivered in this commit

**Web layer (TDD'd, live on the next deploy):**

- `src/lib/mobile/roomplan-capture.ts` — the `RoomPlanCapture` Capacitor plugin
  interface plus:
  - `isRoomPlanScanAvailable()` — the capability the UI gates on. `false` off
    the native shell and on non-LiDAR devices; delegates to the native
    `isSupported()` on a LiDAR device (AC #3, #4).
  - `scanRoom()` — rejects off-native (never resolves through the no-op web
    proxy); on-native returns the parsed `CapturedRoom` plus the `meshUri`.
  - The `CapturedRoom` / `CapturedSurface` / `CapturedObject` payload types.
- `src/lib/mobile/roomplan-capture.test.ts` — 5 vitest cases covering graceful
  degradation both directions, the off-native scan guard, and the native
  payload parse.

**iOS native source (written + wired into the `App` target, build/verify on a
Mac — see below):**

- `ios/App/App/RoomPlanCapturePlugin.swift` — the `CAPBridgedPlugin`
  (`isSupported`, `scanRoom`).
- `ios/App/App/RoomCaptureController.swift` — the full-screen `RoomCaptureView`
  host that runs one session, exports the USDZ, and returns the payload.

## Xcode wiring — done

Both Swift files are in the **`App` target's Compile Sources** phase
(`ios/App/App.xcodeproj/project.pbxproj`), added programmatically with the
`xcodeproj` Ruby gem — the same approach #172/#173 used — not a hand-edit of
`project.pbxproj`.

> Build the **`App` scheme**, not `-target App`. A bare `-target` build does not
> order transitive SPM package dependencies, so `SecureStoragePlugin` fails with
> `unable to resolve module dependency: 'SwiftKeychainWrapper'`.

`capacitor.config.json` (the `packageClassList`) and `App/public` are gitignored
and regenerated by `npx cap sync ios` — which CI runs in
`ios/App/App/ci_scripts/ci_post_clone.sh` — so a fresh checkout must
`npm ci && npx cap sync ios` before `xcodebuild`. RoomPlan is a **system
framework** (present in both the device and simulator SDKs); no Podfile/SPM
package entry is needed — `import RoomPlan` behind a `#if canImport(RoomPlan)`
guard is enough, and the `@available(iOS 16.0, *)` checks keep it dormant on the
iOS-15 deployment floor.

### Registration — manual, NOT automatic (same trap as #175)

Capacitor 8 only instantiates the plugin classes listed in
`capacitor.config.json`'s `packageClassList`, which is regenerated from the npm
`@capacitor/*` packages — a hand-written, non-npm Swift plugin never lands there,
and the Objective-C `CAP_PLUGIN` macro scan is gone. Compiling the file into the
target is **necessary but not sufficient**. So the instance is registered
manually:

- `ios/App/App/MainViewController.swift` — `capacitorDidLoad()` now also calls
  `bridge?.registerPluginInstance(RoomPlanCapturePlugin())`.

Without it, `registerPlugin("RoomPlanCapture")` on the web side resolves to the
no-op web proxy and `scanRoom` rejects "not implemented".

## Entitlements / Info.plist

RoomPlan uses the camera; `NSCameraUsageDescription` is **already present** in
`ios/App/App/Info.plist` (added for the capture pipeline), so no new key is
required. RoomPlan needs no separate entitlement.

## The capture contract

`scanRoom()` resolves with:

- **`capturedRoomJson`** — `JSONEncoder().encode(CapturedRoom)`. RoomPlan's
  `CapturedRoom` is `Codable`; the web wrapper `JSON.parse`s it into the
  `CapturedRoom` type (walls, doors, windows, openings, objects).
- **`meshUri`** — a `file://` URI to the USDZ the plugin wrote to
  `Documents/room-scans/<uuid>.usdz`. Local-first mirrors how Photos cache blobs
  on device before the upload queue syncs them; **uploading the mesh to Supabase
  and attaching it to the Sketch is a later slice.**

`isSupported()` resolves `{ supported: RoomCaptureSession.isSupported }` (false
on < iOS 16 or non-LiDAR hardware).

## Verify — HITL (physical LiDAR device required)

The capture round-trip — scan → post-process → USDZ export → payload back to the
web app — **cannot be verified by an AFK agent**: RoomPlan does not run in the
simulator (`RoomCaptureSession.isSupported == false` there), so this needs a
LiDAR device (iPad Pro / iPhone Pro), Xcode signing, and manual on-device
testing (issue #863 AC #5). The web-side capability + guard behavior (AC #3, #4)
is already covered by `roomplan-capture.test.ts`.
